Shipped work & open builds

Two production side projects, plus three system designs in progress — extensions of patterns I use at work, built out as public proof of concepts.

Contract Clause Analyzer

Shipped
ASP.NET Core · GPT-4 · PostgreSQL · React · Docker

Parses uploaded contract PDFs, extracts individual clauses, and flags high-risk terms — uncapped liability, indemnity overreach, auto-renewal traps — cutting manual review to under 60 seconds per contract.

Async extraction pipeline with GPT-4 prompt orchestration and PostgreSQL persistence for full clause history. Containerized with Docker, structured for future approval-flow and audit-trail integration.

StreamCore

Shipped
ASP.NET Core · React · Azure Blob Storage · Redis · Hangfire

A video learning platform with a custom binary streaming service using HTTP byte-range requests for progressive playback and chunk-level seeking without full re-download.

Background ingestion via Hangfire, scalable media storage on Azure Blob, and Redis caching to cut latency under concurrent load.

Fleet Compliance Event Bus

Concept
ASP.NET Core · RabbitMQ · PostgreSQL Outbox · Docker

Public write-up of the Outbox Pattern system built at Fendahl, generalized into an open reference implementation: transactional outbox, retry policy, dead-letter queue, and a live dashboard for event delivery health.

Goal: give backend teams a drop-in pattern for reliable event delivery without a distributed transaction.

Commodity Settlement Ledger

Concept
ASP.NET Core · CQRS · PostgreSQL · Redis

A CQRS reference build modeling trade settlement: a write model that enforces ledger invariants, a read model optimized and cached in Redis for trader-facing dashboards, and an event log that makes every balance change replayable.

Designed to mirror the auction-and-settlement systems built for biofuel trading, stripped down to a teachable open build.
